# Semana 16 — Desarrollo del proyecto final

**Duración:** 5 días  
**Horas diarias:** 6  
**Total semanal:** 30 horas  

---

## Objetivo de la semana

Implementar y consolidar el **sistema de software diseñado** durante las semanas anteriores, aplicando conceptos de:

- Arquitectura modular y basada en servicios  
- Microservicios y comunicación mediante APIs REST  
- Contenedores Docker para entornos reproducibles  
- Testing profesional (unitario, integración y sistema)  
- Optimización de rendimiento  
- Documentación técnica completa  

Al finalizar la semana, el estudiante podrá **integrar todos los conocimientos del semestre** y avanzar hacia la finalización del proyecto en la Semana 17.

---

## Habilidades que se desarrollarán

- Implementación de microservicios siguiendo principios SOLID y patrones de diseño  
- Construcción de APIs funcionales entre servicios  
- Despliegue de servicios en contenedores Docker  
- Ejecución de testing completo (unitario, integración y sistema)  
- Optimización inicial de rendimiento y flujo de datos  
- Documentación técnica y diagramado profesional de la arquitectura  
- Preparación del sistema para revisión final y entrega de portafolio  

---

## Estructura diaria

**Bloque 1 — 11:00 a 13:00**  
Implementación de componentes y microservicios.  

**Bloque 2 — 16:00 a 19:00**  
Testing, optimización, integración y documentación.  

---

## Día 1 — Preparación de entorno y revisión de arquitectura

**Bloque 1 — Conceptos**  

- Revisión de la arquitectura diseñada  
- Confirmación de la división en microservicios  
- Principios SOLID aplicados a cada servicio  
- Identificación de dependencias y flujo de datos  

**Actividad:**  
Configurar repositorio, entornos locales y contenedores Docker para desarrollo.

**Bloque 2 — Ejercicios**  

- Inicializar contenedores para cada microservicio  
- Documentar endpoints iniciales y responsabilidades  
- Verificar conexión básica entre servicios  

---

## Día 2 — Implementación de microservicios y APIs

**Bloque 1 — Conceptos**  

- Desarrollo de microservicios principales  
- Definición de APIs REST entre servicios  
- Manejo de errores y validaciones iniciales  

**Actividad:**  
Codificar los servicios principales según diseño, asegurando independencia de componentes.

**Bloque 2 — Ejercicios**  

- Implementar endpoints funcionales  
- Testear llamadas API internas  
- Registrar flujos de datos y dependencias en documentación  

---

## Día 3 — Testing profesional y flujo completo

**Bloque 1 — Conceptos**  

- Testing unitario para cada microservicio  
- Pruebas de integración entre servicios  
- Preparación de pruebas de sistema  

**Actividad:**  
Implementar tests automáticos para todos los servicios desarrollados.

**Bloque 2 — Ejercicios**  

- Ejecutar pruebas unitarias y de integración  
- Detectar errores y registrar incidencias  
- Ajustar servicios para garantizar estabilidad del sistema  

---

## Día 4 — Optimización y monitoreo

**Bloque 1 — Conceptos**  

- Análisis de rendimiento de microservicios  
- Profiling de aplicaciones y endpoints  
- Preparación de métricas básicas para observabilidad  

**Actividad:**  
Optimizar consultas, procesos y flujo de datos entre servicios.

**Bloque 2 — Ejercicios**  

- Ejecutar profiling y ajustar rendimiento  
- Configurar logging estructurado en cada servicio  
- Documentar mejoras de optimización y métricas de desempeño  

---

## Día 5 — Documentación final y consolidación del proyecto

**Bloque 1 — Conceptos**  

- Documentación completa de arquitectura  
- Diagramas de microservicios, APIs y contenedores  
- Estrategia de testing final y métricas de observabilidad  

**Actividad:**  
Consolidar todo el proyecto para entrega de portafolio.

**Bloque 2 — Ejercicios**  

- Revisar consistencia de endpoints, servicios y contenedores  
- Registrar testing, optimización y documentación final  
- Preparar base sólida para la **Semana 17 — Finalización del proyecto**  

---

## Entregables de la semana

- Repositorio actualizado con microservicios funcionales  
- APIs documentadas  
- Contenedores Docker configurados para cada servicio  
- Pruebas unitarias, integración y de sistema completas  
- Diagramas de arquitectura y flujos de datos  
- Documentación técnica profesional consolidada  

---

## Conexión con el semestre

Esta semana asegura que el estudiante:

- Integra todos los conocimientos del **Semestre 5**  
- Aplica patrones de diseño, SOLID y testing profesional  
- Prepara un sistema modular listo para finalización y presentación  
- Avanza hacia la **Semana 17 — Finalización y documentación para portafolio profesional**
