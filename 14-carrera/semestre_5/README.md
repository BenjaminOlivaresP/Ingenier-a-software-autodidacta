# Semestre 5 — Arquitectura de Software

**Duración:** 16 semanas  
**Horas semanales:** 30  
**Total aproximado:** 480 horas

---

# Objetivo del semestre

Este semestre se enfoca en el **diseño de sistemas complejos** y en la comprensión profunda de la **arquitectura de software moderna**.

Después de haber desarrollado aplicaciones full stack completas en el semestre anterior, el estudiante comienza a analizar **cómo se diseñan sistemas a gran escala**, cómo se organizan sus componentes y cómo se optimizan para mantener **rendimiento, mantenibilidad y escalabilidad**.

Durante este semestre el estudiante desarrollará habilidades en:

- diseño de **arquitecturas de software robustas**
- aplicación de **patrones de diseño**
- desarrollo de **sistemas distribuidos**
- construcción de **microservicios**
- testing profesional y aseguramiento de calidad
- optimización de software y análisis de rendimiento
- uso de contenedores y entornos reproducibles

El objetivo es que el estudiante evolucione de **desarrollador full stack** a **ingeniero capaz de diseñar sistemas completos**.

---

# Tecnologías utilizadas

- **Docker** y contenedores
- herramientas de **testing profesional**
- arquitectura basada en **servicios**
- herramientas modernas de desarrollo backend
- control de versiones avanzado con **Git / GitHub**

# Proyecto final del semestre

**Aplicación:** Sistema de software diseñado con arquitectura modular o basada en servicios.

**Características posibles:**

- sistema dividido en múltiples servicios
- comunicación entre servicios mediante APIs
- uso de contenedores para ejecución
- testing automatizado
- arquitectura modular clara
- documentación técnica del sistema

**Tecnologías posibles:**

- Node.js o backend moderno
- APIs REST
- Docker
- testing frameworks
- herramientas de monitoreo

**Objetivo del proyecto:**

Diseñar e implementar un sistema de software con una arquitectura clara, mantenible y escalable que demuestre comprensión de principios de ingeniería de software profesional.

---

# Resultados esperados al finalizar el semestre

Al completar el semestre el estudiante será capaz de:

- diseñar **arquitecturas de software modernas**
- aplicar **patrones de diseño**
- construir sistemas basados en **servicios**
- implementar **microservicios básicos**
- trabajar con **contenedores Docker**
- implementar **testing profesional**
- optimizar software y analizar rendimiento

Este semestre prepara al estudiante para el **Semestre 6 — Ingeniería de Sistemas Escalables**, donde se profundizará en **cloud computing, despliegue profesional, seguridad avanzada y construcción de sistemas listos para producción**.
