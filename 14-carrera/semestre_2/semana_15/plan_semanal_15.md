# Semana 15 — Proyecto Final Avanzado

Duración: 5 días  
Horas diarias: 6  
Total semanal: 30 horas

---

# Objetivo de la semana

Construir y consolidar la aplicación web final del semestre.  
Esta semana se enfoca en integrar todos los conocimientos de JavaScript avanzado, asincronía, manipulación del DOM y consumo de APIs para crear una aplicación funcional y dinámica.  

El proyecto permitirá al estudiante:

- aplicar JavaScript avanzado en un proyecto real  
- integrar datos dinámicos y asincronía  
- practicar buenas prácticas de código y estructura  
- preparar un proyecto listo para portafolio

---

# Habilidades que se desarrollarán

Al finalizar la semana, el estudiante podrá:

- estructurar código JavaScript modular y limpio  
- consumir APIs y manejar datos dinámicos  
- manipular el DOM de forma eficiente  
- implementar funcionalidades completas en una aplicación web  
- documentar todo el proyecto de manera profesional

---

# Estructura diaria

Bloque 1 — 11:00 a 13:00  
Planificación, revisión y codificación central.

Bloque 2 — 16:00 a 19:00  
Integración de funcionalidades y pruebas.

---

# Día 1 — Planeación del proyecto

## Bloque 1 — Planificación

Temas:

- definición de funcionalidades  
- estructura de archivos y modularización  
- diagramas de flujo y pseudocódigo  

Actividad:

- Escribir un documento: “Plan de desarrollo de la aplicación”  
- Crear estructura inicial del proyecto

---

## Bloque 2 — Inicio de codificación

- Configurar HTML básico del proyecto  
- Preparar hojas de estilo CSS iniciales  
- Crear archivo principal de JavaScript

---

# Día 2 — Funcionalidades principales

## Bloque 1 — Implementación de lógica

- Funciones principales de la aplicación  
- Manejo de arrays y objetos  
- Validaciones de datos

## Bloque 2 — Manipulación del DOM

- Mostrar datos dinámicamente en la interfaz  
- Actualizar contenido en respuesta a acciones del usuario

---

# Día 3 — Asincronía y APIs

## Bloque 1 — Consumo de APIs

- Integración de APIs externas  
- Lectura de datos y manejo de errores  
- Visualización de información en la aplicación

## Bloque 2 — Async / Await

- Uso de promesas para operaciones asincrónicas  
- Mejoras en la interacción con el usuario

---

# Día 4 — Mejora de UX y estilo

## Bloque 1 — CSS avanzado y diseño

- Layout más pulido  
- Tipografía y colores consistentes  
- Feedback visual para acciones del usuario

## Bloque 2 — Pruebas y depuración

- Revisar errores de lógica  
- Validar la consistencia de la aplicación  
- Optimizar funciones y eventos

---

# Día 5 — Entrega final y documentación

## Bloque 1 — Preparación para entrega

- Revisar estructura de archivos  
- Comentar y documentar código  
- Crear README con instrucciones de uso

## Bloque 2 — Subida al portafolio

- Subir proyecto a GitHub  
- Verificar funcionamiento en distintos navegadores  
- Revisar que todo esté listo para mostrar

---

# Entregables de la semana

- Código completo de la aplicación  
- Documentación profesional del proyecto  
- README con instrucciones  
- Proyecto funcional en GitHub listo para portafolio

---

# Evaluación personal

Responder:

1. ¿Está mi código modular y limpio?  
2. ¿Todas las funcionalidades planeadas funcionan correctamente?  
3. ¿La aplicación es interactiva y dinámica?  
4. ¿He documentado y subido el proyecto al portafolio?  

Si la respuesta es **sí** a la mayoría, el proyecto final del semestre está completo y listo.
