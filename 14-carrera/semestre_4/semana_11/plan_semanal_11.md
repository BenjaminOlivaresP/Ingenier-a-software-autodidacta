# Semana 11 — Testing y validación full stack

Duración: 5 días  
Horas diarias: 6  
Total semanal: 30 horas

---

# Objetivo de la semana

Aprender a probar y validar aplicaciones full stack completas para garantizar que funcionen correctamente, integrando frontend y backend.  
El estudiante desarrollará habilidades para detectar errores, validar datos y asegurar la calidad integral del software.

Durante la semana se estudiará:

- testing unitario y de integración en backend  
- pruebas de frontend y comportamiento de UI  
- pruebas de endpoints y APIs  
- validación de datos y manejo de errores  
- automatización de pruebas full stack  

---

# Habilidades que se desarrollarán

Al finalizar la semana, el estudiante podrá:

- escribir pruebas unitarias e integración en backend y frontend  
- validar la comunicación entre frontend y backend  
- detectar errores y fallos de integración  
- automatizar tests para agilizar el desarrollo  
- documentar resultados de pruebas  
- asegurar que la aplicación cumpla con los requisitos
