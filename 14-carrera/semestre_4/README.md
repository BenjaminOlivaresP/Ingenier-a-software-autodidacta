# Semestre 4 — Sistemas y Desarrollo Full Stack

**Duración:** 16 semanas  
**Horas semanales:** 30  
**Total aproximado:** 480 horas

---

# Objetivo del semestre

Este semestre se centra en la integración de **frontend y backend** para construir aplicaciones **full stack completas** y funcionales.  
El estudiante desarrollará competencias para manejar datos, servicios, autenticación, seguridad y despliegue, creando aplicaciones web modernas listas para producción.

Durante el semestre, el estudiante adquirirá habilidades en:

- desarrollo avanzado de servidores y APIs con **Node.js**  
- integración de frontend dinámico (React, Vue o Vanilla JS avanzado)  
- manejo de **bases de datos relacionales y no relacionales**  
- construcción de endpoints RESTful y consumo de APIs internas y externas  
- autenticación con **JWT** y gestión de roles  
- seguridad básica y buenas prácticas web  
- pruebas, optimización y despliegue en entornos locales y en la nube  

El objetivo es que el estudiante evolucione de aplicaciones backend aisladas a sistemas completos que gestionen **datos, servicios y funcionalidades integradas**.

---

# Tecnologías utilizadas

- **Node.js avanzado**  
- **Express** y frameworks backend  
- **Bases de datos relacionales (SQL)** y no relacionales (NoSQL)  
- **Frontend dinámico** (React, Vue o Vanilla JS avanzado)  
- **Autenticación JWT y control de acceso**  
- **DevOps básico:** despliegue local y en la nube  
- Herramientas de control de versiones: **Git / GitHub**

# Contenido del semestre

### Semana 1 — Repaso de Node.js y Express avanzado  
- Servidores y rutas avanzadas  
- Middleware y control de flujo  
- Programación asincrónica y promesas  

### Semana 2 — Integración frontend-backend  
- Comunicación entre cliente y servidor  
- Consumo de APIs desde el frontend  
- Manejo de estados y sincronización de datos  

### Semana 3 — Estructuras de datos y algoritmos intermedios  
- Arrays, listas, colas, pilas  
- Búsqueda y ordenamiento

- Creación de tokens seguros  
- Middleware de autenticación  
- Roles y permisos de usuario  

### Semana 5 — Seguridad en aplicaciones web  
- Protección de endpoints  
- Sanitización de datos y prevención de inyecciones  
- HTTPS y buenas prácticas de seguridad  

### Semana 6 — Bases de datos avanzadas y relaciones complejas  
- Modelado relacional avanzado  
- Consultas complejas y optimizadas  
- Relaciones 1:N, N:N y anidamiento de documentos  

### Semana 7 — ORM y manejo de datos con frameworks  
- Uso de **Sequelize**, **TypeORM** o **Mongoose**  
- Migraciones y sincronización de modelos  
- Operaciones CRUD avanzadas  

### Semana 8 — Consumo de APIs internas y externas  
- Integración con servicios externos  
- Fetch y Axios  
- Manejo de errores y validación de respuestas  

### Semana 9 — CRUD avanzado y transacciones  
- Operaciones complejas con múltiples tablas o colecciones  
- Transacciones y atomicidad  
- Validaciones y consistencia de datos  

### Semana 10 — Optimización y rendimiento de aplicaciones  
- Profiling y análisis de rendimiento  
- Caching y paginación  
- Estrategias de escalabilidad básica  

### Semana 11 — Testing y validación full stack  
- Pruebas unitarias, de integración y end-to-end  
- Testing de frontend y backend  
- Automatización de pruebas y reportes  

### Semana 12 — Despliegue en entornos locales y remotos  
- Configuración de servidores locales  
- Despliegue en la nube (Heroku, Vercel o similares)  
- Variables de entorno y entornos de prueba  

### Semana 13 — Integración continua y control de versiones avanzado  
- Uso avanzado de Git y ramas de desarrollo  
- Integración con pipelines de CI/CD básicos  
- Pruebas automáticas y despliegue continuo  

### Semana 14 — Proyecto intermedio: Aplicación full stack funcional  
- Desarrollo de un proyecto completo integrando frontend y backend  
- Conexión a bases de datos y APIs externas  
- Implementación de autenticación y roles  

### Semana 15 — Desarrollo del proyecto final  
- Consolidación de funcionalidades  
- Optimización de rendimiento y seguridad  
- Preparación de documentación y guía de usuario  

### Semana 16 — Finalización y presentación del proyecto  
- Revisión completa del proyecto  
- Pruebas finales y corrección de errores  
- Presentación profesional del sistema full stack
