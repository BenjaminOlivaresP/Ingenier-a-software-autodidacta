# Semana 9 — APIs RESTful: diseño y consumo

Duración: 5 días  
Horas diarias: 6  
Total semanal: 30 horas

---

# Objetivo de la semana

Aprender a diseñar y consumir APIs RESTful, construyendo endpoints claros y estructurados que permitan la interacción de aplicaciones con el backend.  
El estudiante desarrollará la capacidad de crear servicios web accesibles y de integrarlos con aplicaciones externas o frontend.

Durante la semana se revisarán:

- principios REST y arquitectura de APIs  
- rutas y métodos HTTP (GET, POST, PUT, DELETE)  
- estructuras de respuesta y manejo de status codes  
- consumo de APIs desde frontend y herramientas externas  
- documentación de endpoints  

---

# Habilidades que se desarrollarán

Al finalizar la semana, el estudiante podrá:

- diseñar APIs RESTful siguiendo buenas prácticas  
- implementar rutas con Node.js y Express  
- enviar y recibir datos en formato JSON  
- manejar status codes y mensajes de respuesta  
- consumir APIs desde aplicaciones externas o frontend  

---

# Estructura diaria

Bloque 1 — 11:00 a 13:00  
Teoría sobre REST, métodos HTTP y diseño de APIs.

Bloque 2 — 16:00 a 19:00  
Práctica implementando endpoints y consumiendo APIs con fetch o Postman.

---

# Día 1 — Fundamentos de APIs RESTful

## Bloque 1 — Teoría

- conceptos de API y REST  
- diferencias entre REST y SOAP  
- métodos HTTP y sus usos  
- estructura de rutas y recursos  

Actividad:  
Documento: “Principios de diseño de APIs RESTful”.

## Bloque 2 — Ejercicios

- diagramar recursos de ejemplo para una API de usuarios y tareas  
- asignar métodos HTTP a cada operación  

---

# Día 2 — Creación de endpoints con Node.js y Express

## Bloque 1 — Teoría

- estructura básica de rutas en Express  
- uso de router y modularización de endpoints  
- envío de respuestas JSON  

Actividad:  
Documento: “Endpoints con Express”.

## Bloque 2 — Ejercicios

- crear endpoints GET y POST para un recurso simple  
- probar respuestas con Postman o Insomnia  

---

# Día 3 — Manejo de status codes y validaciones

## Bloque 1 — Teoría

- códigos de estado HTTP: 200, 201, 400, 404, 500  
- validación de datos de entrada  
- mensajes de error claros y consistentes  

## Bloque 2 — Ejercicios

- agregar validaciones a endpoints existentes  
- enviar status codes adecuados según la situación  
- probar casos de error y éxito  

---

# Día 4 — Consumo de APIs desde frontend y herramientas externas

## Bloque 1 — Conceptos

- uso de fetch o Axios para consumir APIs  
- integración con aplicaciones web dinámicas  
- pruebas con Postman o Insomnia  

## Bloque 2 — Ejercicios

- crear pequeños scripts frontend para consumir los endpoints creados  
- mostrar datos en consola o en elementos HTML  

---

# Día 5 — Mini-proyecto de API funcional

## Bloque 1 — Integración

- consolidar todos los endpoints de la semana  
- documentar cada ruta y su funcionamiento  
- probar la API con frontend o herramientas externas  

## Bloque 2 — Buenas prácticas y revisión

- modularizar rutas y controladores  
- validar consistencia de respuestas  
- preparar documentación profesional de la API
