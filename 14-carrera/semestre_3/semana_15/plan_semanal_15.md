# Semana 15 — Desarrollo del proyecto final

Duración: 5 días  
Horas diarias: 6  
Total semanal: 30 horas

---

# Objetivo de la semana

Construir el proyecto final del semestre: un sistema backend completo con API funcional, integrando todos los conocimientos previos de Node.js, Express y SQL.  
El estudiante desarrollará una aplicación robusta, modular y documentada lista para pruebas y futura integración con frontend.

Durante la semana se trabajará en:

- diseño completo de la API final  
- implementación de todos los endpoints CRUD  
- manejo avanzado de errores  
- integración de autenticación básica  
- pruebas completas de la API  
- documentación profesional del proyecto  

---

# Habilidades que se desarrollarán

Al finalizar la semana, el estudiante podrá:

- construir un backend completo y funcional  
- implementar endpoints RESTful seguros y robustos  
- manejar errores y registros de manera profesional  
- documentar la API para otros desarrolladores  
- preparar el proyecto final para demostración y portafolio  

---

# Estructura diaria

Bloque 1 — 11:00 a 13:00  
Diseño, desarrollo e implementación de endpoints.

Bloque 2 — 16:00 a 19:00  
Pruebas, ajustes, autenticación y documentación.

---

# Día 1 — Planificación del proyecto final

## Bloque 1 — Diseño

Temas:

- definir funcionalidades principales  
- identificar recursos y endpoints necesarios  
- planificar rutas, métodos y estructura de la base de datos  

Actividad:

Documento:  
"Planificación del proyecto final: diagrama de endpoints y base de datos"

---

## Bloque 2 — Configuración inicial

Práctica:

- crear repositorio y estructura de carpetas  
- instalar dependencias necesarias (Node.js, Express, SQL drivers)  
- configurar conexión inicial con base de datos  

---

# Día 2 — Implementación CRUD inicial

## Bloque 1 — CREATE y READ

Temas:

- endpoints POST y GET  
- validación de datos de entrada  
- pruebas con Postman  

## Bloque 2 — Ejercicios

Práctica:

- implementar rutas para crear y leer datos  
- conectar con la base de datos  
- verificar resultados y corregir errores  

---

# Día 3 — Implementación UPDATE y DELETE

## Bloque 1 — Conceptos

Temas:

- endpoints PUT y DELETE  
- manejo de errores y consistencia de datos  
- seguridad básica en operaciones  

## Bloque 2 — Práctica

- implementar rutas para actualizar y eliminar registros  
- probar operaciones con datos reales  
- refactorizar código para modularidad y claridad  

---

# Día 4 — Autenticación y pruebas finales

## Bloque 1 — Autenticación

Temas:

- autenticación básica con JWT  
- protección de endpoints sensibles  
- manejo de tokens y sesiones  

## Bloque 2 — Pruebas completas

- ejecutar pruebas integrales de todos los endpoints  
- registrar errores y solucionarlos  
- verificar consistencia de la base de datos y seguridad  

---

# Día 5 — Documentación y preparación del portafolio

## Bloque 1 — Documentación profesional

- crear README completo del proyecto  
- incluir lista de endpoints, métodos, parámetros y ejemplos  
- capturas de pantalla y flujo de pruebas  

## Bloque 2 — Consolidación final

- revisión completa del código y estructura del proyecto  
- pruebas finales de funcionalidad y consistencia  
- preparar entrega final para revisión y portafolio  

---

# Entregables de la semana

- Sistema backend completo con API funcional  
- Base de datos SQL conectada y operativa  
- Endpoints CRUD con autenticación básica  
- Documentación profesional del proyecto final  
- Pruebas integrales y flujo de operaciones validado  

---

# Conexión con el semestre

Esta semana es la culminación de todo lo aprendido, preparando al estudiante para:

- despliegue y pruebas locales (Semana 16)  
- integración futura con frontend y aplicaciones full stack  
- creación de portafolio profesional con proyectos completos
