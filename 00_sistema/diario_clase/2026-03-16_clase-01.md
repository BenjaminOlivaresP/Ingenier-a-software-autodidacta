# Materia bloque 1

## Algoritmo y su función con ejemplo simple

**Problema:**  
Calcular el promedio de tres números.

Un humano podría hacerlo mentalmente, pero para programar un computador se necesitan pasos exactos.

**Pasos lógicos:**

- pedir el primer número  
- pedir el segundo número  
- pedir el tercer número  
- sumar los tres números  
- dividir el resultado entre 3  
- mostrar el resultado  

Esto ya constituye **un algoritmo**.

---

## Forma de escribir algoritmos (pseudocódigo)

Los programadores suelen escribir primero **pseudocódigo**.  
Este es un lenguaje intermedio entre el español y un lenguaje de programación.

### Ejemplo

```
INICIO

Leer numero1
Leer numero2
Leer numero3

suma = numero1 + numero2 + numero3

promedio = suma / 3

mostrar promedio

FIN
```

Esto no es código ejecutable todavía, pero **ya es lógica programable**.

# Algoritmos básicos

## Ejercicio 1 — Determinar si un número es par o impar

**Problema:**  
Determinar si un número ingresado por el usuario es par o impar.

### Pseudocódigo

```
INICIO

leer numero

si numero % 2 == 0 entonces
    mostrar "El número es par"
si no
    mostrar "El número es impar"

FIN
```

---

## Ejercicio 2 — Desafío 1: Área de un rectángulo

Recordando que:

```
A = b × h
```

Donde:

- **A** = área  
- **b** = base  
- **h** = altura  

Pero un computador entiende **pasos**, no fórmulas.

**Pasos lógicos:**

- pedir la base  
- pedir la altura  
- multiplicar base × altura  
- mostrar el resultado  

Esto también es **un algoritmo**.

---

## Nota 1

Es importante entender que **todo el software del mundo funciona así**.

Aplicaciones como:

- Instagram  
- sistemas bancarios  
- videojuegos  
- inteligencia artificial  
- sistemas operativos  

funcionan gracias a **miles o millones de algoritmos ejecutándose**.

Un programador es realmente:

> un **diseñador de algoritmos para resolver problemas**.

## Ejercicio 3 — Desafío 2: ¿Cuál número es mayor?

Determinar cuál número es mayor entre dos números.

### Datos de entrada

- número 1  
- número 2  

### Operación

Compararlos.

### Resultado

Mostrar cuál es mayor.

### Algoritmo en pseudocódigo

```
INICIO

Leer numero1
Leer numero2

si numero1 > numero2 entonces
    mostrar "numero1 es mayor"

si numero2 > numero1 entonces
    mostrar "numero2 es mayor"

si numero1 == numero2 entonces
    mostrar "los números son iguales"

FIN
```

---

## Nota 2

Estos ejercicios corresponden a **lógica condicional**.

En programación se conocen como **estructuras de control**.

Las principales son:

- **Secuencia** (pasos normales)  
- **Condiciones** (si pasa algo, hacer algo)  
- **Repeticiones** (hacer algo muchas veces)

Esta es **la base absoluta de todos los lenguajes de programación**.

---

## Nota 3

Un computador **no piensa**.

Simplemente ejecuta instrucciones extremadamente rápido.

Por eso los programadores deben diseñar **algoritmos claros y correctos**.

## HTML (estructura de la página)

Se creó el archivo:

```
index.html
```

Este archivo define **la estructura de la página web**, incluyendo:

- encabezado
- secciones de contenido
- texto
- listas
- pie de página

También se aprendieron nuevas etiquetas como:

- `ul` → lista desordenada  
- `li` → elemento de lista  

Estas estructuras se utilizan comúnmente en:

- menús de navegación  
- listas de contenido  
- interfaces web  

---

## CSS (estilos visuales)

Luego se creó el archivo:

```
style.css
```

El cual se conectó al HTML mediante:

```
<link rel="stylesheet" href="style.css">
```

CSS permite controlar:

- colores  
- fuentes  
- márgenes  
- espaciado  
- diseño visual de la página  

Se realizaron mejoras como:

- fondo de página  
- bloque central de contenido  
- alineación del encabezado  
- separación de elementos de lista

Esto permite trabajar de manera más rápida y visual durante el desarrollo web.
