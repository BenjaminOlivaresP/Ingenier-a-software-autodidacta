# Método de estudio — Ingeniería de Software Autodidacta

Autor: Benjamín Olivares

Este documento define el sistema de aprendizaje utilizado en esta carrera autodidacta.

## Principios

1. Aprender construyendo.
2. Resolver problemas reales.
3. Documentar el proceso de aprendizaje.
4. Progresar con proyectos pequeños y constantes.

---

## Distribución de aprendizaje

70% Micro-proyectos  
20% Ejercicios técnicos  
10% Proyectos grandes

### Micro-proyectos
Pequeños proyectos que permiten aprender rápidamente conceptos nuevos.

Ejemplos:
- pequeñas páginas web
- scripts simples
- experimentos con código

### Ejercicios técnicos
Problemas enfocados en lógica y pensamiento computacional.

Ejemplos:
- algoritmos simples
- pseudocódigo
- ejercicios de programación

### Proyectos grandes
Integración de múltiples conocimientos en un solo proyecto.

Ejemplos:
- páginas web completas
- aplicaciones
- sistemas simples

---

## Horario diario

Bloque 1 — 11:00 a 13:00  
Fundamentos, teoría y pensamiento computacional.

Bloque 2 — 16:00 a 19:00  
Programación, práctica y construcción de proyectos.

---

## Objetivo final

Convertirme en un ingeniero de software autodidacta capaz de:

- desarrollar aplicaciones
- crear sistemas web
- comprender arquitectura de software
- resolver problemas complejos
