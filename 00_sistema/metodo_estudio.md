# Método de Estudio — Ingeniería de Software Autodidacta

Autor: Benjamín Ignacio Olivares Pareja  
Inicio del programa: Marzo 2026

Este documento define la metodología oficial de aprendizaje utilizada
en esta carrera autodidacta de ingeniería de software.

El objetivo de este sistema es desarrollar conocimiento técnico profundo
mediante una combinación equilibrada de teoría, práctica y construcción
de software real.

---

# Filosofía del aprendizaje

La programación no se aprende únicamente leyendo o viendo cursos.

Se aprende principalmente mediante:

- construcción de proyectos
- resolución de problemas
- práctica constante
- reflexión sobre el proceso

Por esta razón el sistema prioriza la construcción práctica por sobre
la teoría pasiva.

---

# Principios del sistema

1. Aprender construyendo.
2. Resolver problemas reales.
3. Documentar el proceso de aprendizaje.
4. Progresar con proyectos pequeños y constantes.
5. Desarrollar pensamiento computacional.
6. Comprender cómo funcionan los sistemas.

---

# Modelo pedagógico

El sistema utiliza un modelo de **aprendizaje intercalado**.

Esto significa que durante el día se trabajan distintos tipos de
habilidades cognitivas en lugar de concentrarse en un solo tema.

Esto permite:

- mejorar la retención de conocimiento
- evitar la fatiga mental
- reforzar conexiones entre conceptos
- desarrollar pensamiento flexible

---

# Distribución de aprendizaje

70% Micro-proyectos  
20% Ejercicios técnicos  
10% Proyectos grandes

Este modelo prioriza la práctica constante sobre el estudio pasivo.

---

# Micro-proyectos

Pequeños proyectos diseñados para aprender conceptos específicos
de manera rápida.

Permiten experimentar, cometer errores y aprender sin el peso
de proyectos grandes.

Ejemplos:

- pequeñas páginas web
- scripts simples
- experimentos con código
- componentes interactivos
- visualizaciones de datos

---

# Ejercicios técnicos

Problemas enfocados en desarrollar pensamiento computacional
y habilidades de programación.

Ejemplos:

- algoritmos simples
- pseudocódigo
- manipulación de datos
- ejercicios de lógica

---

# Proyectos grandes

Los proyectos grandes integran múltiples conocimientos
y simulan el desarrollo de software real.

Estos proyectos permiten desarrollar:

- arquitectura de software
- organización de código
- resolución de problemas complejos
- disciplina de desarrollo

Ejemplos:

- aplicaciones web completas
- herramientas digitales
- sistemas interactivos

---

Duración diaria: 6 horas

Horario base:

11:00 – 13:00

16:00 – 19:00

---

# Arquitectura cognitiva del día

El día de estudio se divide en tres modos de aprendizaje.

### 1. Activación conceptual

11:00 – 12:00

Estudio de fundamentos:

- historia de la computación
- conceptos de programación
- arquitectura de sistemas
- teoría del software

Este bloque desarrolla comprensión profunda del campo.

---

### 2. Pensamiento computacional

12:00 – 13:00

Ejercicios de lógica y algoritmos:

- pseudocódigo
- resolución de problemas
- diseño de soluciones

Este bloque fortalece la capacidad de pensar como programador.

---

### 3. Ingeniería práctica

16:00 – 19:00

Construcción de software real.

Durante este bloque se desarrollan:

- micro-proyectos
- ejercicios prácticos
- proyectos del semestre

Este bloque representa la mayor parte del aprendizaje.

---

# Sistema de documentación

Todo el proceso de aprendizaje se documenta mediante tres herramientas.

### Diario de ingeniería

Registro diario o semanal del aprendizaje.

Permite:

- reflexionar sobre problemas
- consolidar conocimientos
- registrar descubrimientos técnicos

---

### Sistema de progreso

Seguimiento del avance en:

- semanas del semestre
- habilidades técnicas
- proyectos completados

---

### Repositorio de código

Todo el trabajo se guarda en repositorios utilizando
control de versiones con Git.

Esto permite:

- registrar evolución del aprendizaje
- documentar proyectos
- construir portafolio profesional

---

# Mentalidad del sistema

El objetivo de esta carrera autodidacta no es únicamente
aprender a programar.

El objetivo es desarrollar la mentalidad de un ingeniero:

- observar sistemas
- descomponer problemas
- construir soluciones
- mejorar continuamente

---

# Objetivo final

Convertirme en un ingeniero de software autodidacta capaz de:

- desarrollar aplicaciones
- crear sistemas web
- comprender arquitectura de software
- resolver problemas complejos
- construir herramientas que generen impacto real
